# Phase 4.1 Plan 01: CheckInEntry Response Shape Fix Summary

Server returns CheckInEntry-compatible JSON from GET /events/{id}/check-in; TeamRolesTable.userId made nullable+SET_NULL matching V8 migration.

## Objective

Fix two P0 server-side data mismatches that caused an empty member response list and potential NPE on member removal queries.

## Tasks Completed

| 2 | Fix TeamRolesTable nullable user_id (PARTIAL-01) | b9890d5 | TeamsTable.kt |

## What Was Built

**Task 1:** Added `CheckInEntryResponse`, `AttendanceResponseDto`, `AttendanceRecordDto` server-side DTOs to `AttendanceRepository.kt`. Added `getCheckInEntries(eventId)` to the repository interface and implementation — queries team members for the event (via event_teams → team_roles → users), then loads attendance responses and records separately, merging into the joined DTO shape. Route now responds with `List<CheckInEntryResponse>` matching `List<CheckInEntry>` on the client.

**Task 2:** Changed `TeamRolesTable.userId` from non-nullable CASCADE to `.nullable()` + `ReferenceOption.SET_NULL`, matching the V8 migration. No row-reading call sites needed changes — all team member queries join `UsersTable` and read from `UsersTable.id`.
