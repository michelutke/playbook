2. Update `loadStats` to build the eventTypes map:
```kotlin
fun loadStats(userId: String) {
    viewModelScope.launch {
        attendanceRepository.getRawAttendance(userId).onSuccess { responses ->
            // Build eventTypes map from events the user has responses for
            val eventIds = responses.map { it.eventId }.toSet()
            val eventTypes = mutableMapOf<String, String>()
            eventRepository.getMyEvents().onSuccess { events ->
                events.forEach { ewt ->
                    if (ewt.event.id in eventIds) {
                        eventTypes[ewt.event.id] = ewt.event.type
                    }
                }
            }
            val stats = AttendanceStatsCalculator.calculateStats(responses, eventTypes)
            _state.update { it.copy(presencePct = stats.presencePct) }
        }
    }
}
```

Currently `Screen.Profile -> PlaceholderScreen("Profile")` in AppNavigation.kt. Need to show PlayerProfileScreen with current user's data.

The challenge: PlayerProfileScreen requires `teamId` and `userId` params. For the Profile tab (own profile), we need the current user's ID and a team ID. The user may belong to multiple teams -- for the profile tab, load the first team from roles.

Replace the `Screen.Profile` case in AppNavigation.kt:

```kotlin
Screen.Profile -> {
    val viewModel: PlayerProfileViewModel = viewModel { KoinPlatform.getKoin().get() }
    val userPreferences = KoinPlatform.getKoin().get<ch.teamorg.preferences.UserPreferences>()
    val userId = userPreferences.getUserId() ?: ""
    // Load profile using first available team -- ViewModel handles loading
    LaunchedEffect(userId) {
        if (userId.isNotEmpty()) {
            // Get first team from user's roles
            val teamRepo = KoinPlatform.getKoin().get<ch.teamorg.repository.TeamRepository>()

    PlayerProfileScreen(
        teamId = "", // Will be set by loadProfile

```

Wait -- this is getting complex. Simpler approach: have the ViewModel itself resolve the teamId when called without one. But that changes the ViewModel API.

Better approach: Use a `rememberCoroutineScope` + `remember` to resolve teamId once:

```kotlin
Screen.Profile -> {
    val viewModel: PlayerProfileViewModel = viewModel { KoinPlatform.getKoin().get() }
    val userPreferences = remember { KoinPlatform.getKoin().get<ch.teamorg.preferences.UserPreferences>() }
    val userId = remember { userPreferences.getUserId() ?: "" }
    val teamRepository = remember { KoinPlatform.getKoin().get<ch.teamorg.repository.TeamRepository>() }
    var teamId by remember { mutableStateOf("") }

    LaunchedEffect(userId) {
        if (userId.isNotEmpty()) {
            teamRepository.getMyRoles().onSuccess { roles ->
                teamId = roles.teamRoles.firstOrNull()?.teamId ?: ""
                if (teamId.isNotEmpty()) {
                    viewModel.loadProfile(teamId, userId)
                }
            }
        }
    }

    PlayerProfileScreen(
        teamId = teamId,
        userId = userId,
        viewModel = viewModel,
        onBack = { },
        onLeftTeam = {
            backStack.removeAll { it == Screen.Profile }
            backStack.add(Screen.Teams)
        }
    )
}
```
