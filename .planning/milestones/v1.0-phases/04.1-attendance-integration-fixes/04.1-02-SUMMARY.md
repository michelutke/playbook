# Phase 4.1 Plan 02: Fix Stats emptyMap + Profile Tab Navigation Summary

Fixed two P1 client-side regressions: per-type attendance breakdown always showing 0% due to emptyMap(), and Profile bottom-nav tab routing to PlaceholderScreen instead of PlayerProfileScreen.

## What Was Built

**Task 1 ‚Äî Fix emptyMap in calculateStats:**
- Added `EventRepository` as 5th constructor param to `PlayerProfileViewModel`
- `loadStats()` now builds `eventTypes: Map<String, String>` by calling `eventRepository.getMyEvents()` and filtering to events matching the user's attendance response IDs
- `PlayerProfileState` gains `trainingPresencePct` and `matchPresencePct` fields
- `UiModule` factory updated from `get(), get(), get(), get()` to 5 `get()` calls

**Task 2 ‚Äî Wire Profile tab:**
- `Screen.Profile` case in `AppNavigation.kt` replaced: `PlaceholderScreen("Profile")` ‚Üí full `PlayerProfileScreen` setup
- `userId` resolved from `UserPreferences`, `teamId` resolved from `TeamRepository.getMyRoles()` first team role
- `LaunchedEffect(userId)` calls `viewModel.loadProfile(teamId, userId)` once both IDs are available
- `onLeftTeam` clears Profile from backStack and pushes `Screen.Teams`

## Decisions Made

1. `EventRepository` injected as 5th param ‚Äî straightforward Koin `get()`, no new factory needed
2. Profile tab uses `getMyRoles().teamRoles.firstOrNull()?.teamId` ‚Äî consistent with existing CalendarViewModel pattern; acceptable for MVP where users typically belong to one team
