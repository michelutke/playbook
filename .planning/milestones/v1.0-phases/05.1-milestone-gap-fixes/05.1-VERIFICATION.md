### Human Verification Required

#### 1. Offline RSVP flush on real device foreground

**Test:** Queue an RSVP while offline. Background the app. Restore network. Bring app to foreground.
**Expected:** RSVP syncs to server within seconds of foreground.
**Why human:** `LaunchedEffect(authState)` re-fires on foreground only if `authState` changes (e.g. token refresh) or process death — not guaranteed on simple background/foreground unless platform triggers recomposition. On Android, if the process is alive and `authState` does not change, the `LaunchedEffect` does not re-fire. This may mean the flush only triggers reliably after process death or re-login, not every foreground. Worth testing on device to confirm the trigger semantics match the intended behavior.

#### 2. Team names visible in notification settings UI

**Test:** Log in as a user with team membership, navigate to notification settings.
**Expected:** Team names (e.g. "U14 Boys") appear in the team selector, not UUIDs.
**Why human:** Code path verified but requires a live `ClubRepository` response with populated team data to confirm end-to-end.
