**AttendanceRoutes.kt** — `fun Route.attendanceRoutes()` inside `authenticate("jwt")`:

1. `GET /events/{id}/attendance` — calls `attendanceRepo.getEventAttendance(eventId)`, returns JSON array of response objects
2. `GET /events/{id}/attendance/me` — extracts userId from JWT `call.principal<JWTPrincipal>()!!.payload.subject`, calls `getMyResponse(eventId, userId)`, returns single response or 204
3. `PUT /events/{id}/attendance/me` — receives `{ "status": "confirmed"|"declined"|"unsure", "reason": "..." }`:
   - Validate: if status == "unsure" and reason is blank -> 400 "Reason required for unsure status"
   - Check deadline: `if (attendanceRepo.isDeadlinePassed(eventId)) -> 409 "Response deadline has passed"`
   - Otherwise: `attendanceRepo.upsertResponse(eventId, userId, status, reason)` -> 200
4. `GET /users/{userId}/attendance` — query params `from`, `to`; calls `getRawAttendance(userId, from, to)`
5. `GET /teams/{teamId}/attendance` — query params `from`, `to`; calls `getTeamAttendance(teamId, from, to)`. Requires coach role on team.

**AbwesenheitBackfillJob.kt** — Ktor coroutine-based background job:

```kotlin
class AbwesenheitBackfillJob {
    enum class Status { PENDING, DONE, FAILED }

    // In-memory status per userId (ConcurrentHashMap)
    private val statusMap = ConcurrentHashMap<UUID, Status>()

    fun getStatus(userId: UUID): Status = statusMap[userId] ?: Status.DONE

    suspend fun enqueue(userId: UUID, ruleId: UUID, scope: Application) {
        statusMap[userId] = Status.PENDING
        scope.launch {
            try {
                runBackfill(userId, ruleId)
                statusMap[userId] = Status.DONE
            } catch (e: Exception) {
                logger.error("Backfill failed for user=$userId rule=$ruleId", e)
                statusMap[userId] = Status.FAILED
            }
        }
    }

    private suspend fun runBackfill(userId: UUID, ruleId: UUID) {
        // 1. Load the rule from AbwesenheitRepository
        // 2. Find all future active events for user's teams
        // 3. For each event, check if rule matches:
        //    - recurring: event date's dayOfWeek in rule.weekdays AND (rule.endDate null OR eventDate <= endDate)
        //    - period: eventDate in [rule.startDate, rule.endDate]
        // 4. For matching events, bulk insert declined-auto responses
        //    - SKIP where manual_override = true (preserve explicit player responses)
        // 5. Use attendanceRepo.bulkInsertAutoDeclines(ruleId, userId, eventUserPairs)
    }
}
```

Register as singleton in Koin (in Koin.kt appModule): `single { AbwesenheitBackfillJob() }`

**AutoPresentJob.kt** — Periodic background job (similar to EventMaterialisationJob pattern):

```kotlin
fun Application.startAutoPresentJob() {
    launch {
        while (true) {
            delay(15.minutes) // check every 15 minutes
            try {
                runAutoPresentCheck()
            } catch (e: Exception) {
                logger.error("Auto-present job failed", e)
            }
        }
    }
}

private suspend fun runAutoPresentCheck() {
    // 1. Find events where end_at < NOW() and check_in_enabled = false and status = 'active'
    // 2. For each: find attendance_responses with status = 'confirmed'
    // 3. For each confirmed response: INSERT attendance_records(status='present', set_by=SYSTEM_UUID)
    //    ON CONFLICT DO NOTHING (don't overwrite coach overrides)
    // 4. Use a "processed" marker to avoid re-processing (e.g., a processed_at column or separate tracking)
}
```
