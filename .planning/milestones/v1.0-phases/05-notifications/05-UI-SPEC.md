## Design Principles

This phase integrates two new surfaces into an established design system.
Extend existing patterns — do not introduce new visual vocabulary.

- **InboxScreen** replaces `PlaceholderScreen("Inbox")` — follows EventListScreen layout: LazyColumn + Card pattern
- **NotificationSettingsScreen** follows the vertical grouped-toggle layout established in AddAbsenceSheet / CoachOverrideSheet
- **Reminder row** on EventDetailScreen follows the existing detail-row pattern (icon + label + trailing tap target)

## Screens & States

### InboxScreen

**Layout:** Scaffold with TopAppBar ("Inbox") + gear icon action (contentDescription = "Notification settings") → NotificationSettingsScreen. LazyColumn of NotificationRows. FloatingActionButton: none.

**States:**

| State | Visual |
|-------|--------|
| Loading | Circular progress indicator centered in list area |
| Empty | Centered empty state: icon `Icons.Outlined.Notifications` + heading "No notifications" + body "You're all caught up." |
| Loaded — has unread | Standard list; unread rows have left-edge `PrimaryBlue` 3dp vertical bar + unread dot |
| Loaded — all read | Standard list; no unread indicators |
| Error | Error card at top: "Could not load notifications. Pull to refresh." |

**NotificationRow anatomy:**
- Leading: notification type icon (16dp, tinted `MutedForegroundDark` if read, `PrimaryBlue` if unread)
- Body: message text (`bodyLarge`) on line 1; metadata line (`labelMedium`, `MutedForegroundDark`): team name + "·" + relative time
- Trailing: unread dot (8dp circle, `PrimaryBlue`) — visible only when unread
- Full row tap: marks read + deep-links to related screen
- Row background: `CardDark` (`#1C1C2E`), `RoundedCornerShape(12.dp)`, 4dp vertical spacing between rows, 16dp horizontal padding

**"Mark all as read" button:** Text button in TopAppBar actions — only visible when there are unread notifications. Label: "Mark all read". `PrimaryBlue` text color.

### NotificationSettingsScreen

**Entry point:** Gear icon in InboxScreen toolbar.

**Layout:** Scaffold with TopAppBar ("Notification Settings") + back arrow. Vertically scrollable Column of grouped setting sections.

**Sections (per team — user selects team via team picker at top):**

1. **Events** — Toggles: "New event", "Event changes", "Cancellations"
2. **Reminders** — Toggle: "Event reminders" (on/off) + when enabled, row: "Default lead time" showing current value (e.g. "2 hours") + tap to open ReminderPickerSheet
3. **Responses** (coach only — hidden for players) — Toggle row with two options: "Per response" / "Pre-event summary" (segmented/radio, not two separate toggles)
4. **Absences** (coach only) — Toggle: "Absence notifications"

Section headers: 12sp `labelMedium`, `MutedForegroundDark`, uppercase, 24dp top padding.
Toggle rows: 48dp minimum height, `bodyLarge` label left, M3 Switch right, 16dp horizontal padding.

### ReminderPickerSheet (BottomSheet)

Triggered from: EventDetailScreen Reminder row, and NotificationSettingsScreen Default lead time row.

**Content:**
- Header: "Set Reminder" (titleLarge)
- Free-form duration input: NumberField (hours) + NumberField (minutes) — two adjacent fields with unit labels
- Preset chips row: "30 min", "1 h", "2 h", "1 day", "2 days" — tap to fill fields
- CTA button: "Set Reminder" (full-width FilledButton, `PrimaryBlue`)
- Secondary: "No reminder" text button below CTA (for per-event override — removes custom reminder, falls back to global default)

### EventDetailScreen — Reminder Row

Position: after location row, before description section.

Row anatomy: `Icons.Outlined.Alarm` (20dp) + "Reminder" label (`bodyLarge`) + trailing: current value text (`labelLarge`, `MutedForegroundDark`) e.g. "2 hours before" or "Global default (2 h)" + `>` chevron icon. Full row tap opens ReminderPickerSheet in per-event mode.

## Interaction Contracts

### Unread badge on bottom nav

- Badge count: integer, shown as pill overlay on Inbox icon
- 0 count: no badge (not "0")
- 1–99: show number
- 100+: show "99+"
- Updates in real-time when InboxViewModel marks notifications as read

### Read-on-tap

- Tapping a NotificationRow immediately marks it read (optimistic, no loading state)
- Then navigates to deep-link destination
- If destination navigation fails: notification still marked read, snackbar: "Could not open. Try again."

### Mark all as read

- Tap "Mark all read" button
- Optimistic: all unread indicators disappear immediately
- On server error: revert unread state + snackbar: "Could not mark read. Try again."

### Notification Settings save

- Toggles save immediately on change (no explicit Save button for toggles)
- Each toggle change: debounce 300ms, then fire PATCH to server
- On error: revert toggle + snackbar: "Settings not saved. Try again."
- Team picker at top: changing team reloads settings for selected team

### ReminderPickerSheet save

- "Set Reminder" button: validates duration > 0, sends update, dismisses sheet
- "No reminder": sends null override for that event, dismisses sheet
- Invalid input (0 duration): "Set Reminder" button disabled, field border turns `DestructiveRed`
