## Root Cause Found

The crash is in **LoginScreen composition** via `kmpViewModel(parameters = ...)`.

When `authState = Unauthenticated`, `LaunchedEffect` fires immediately and sets
`backStack = [Screen.Login]`. NavDisplay renders `LoginScreen`, which calls:
```kotlin
val viewModel: LoginViewModel = kmpViewModel(
    parameters = { parametersOf({ backStack.add(Screen.Events) }) }
)
```

This is Koin with a `() -> Unit` lambda parameter. **Hypothesis**: Koin's
`params.get()` for a `() -> Unit` lambda fails on Kotlin Native due to
type token issues with function types.

## THE FIX (next session)

**Option A ŌĆö Change factory to not use params.get() for lambdas:**

Change `AppNavigation.kt` to NOT pass navigation callbacks through Koin.
Instead, inject the callbacks directly when creating NavEntries:

In `AppNavigation.kt`, the `LoginScreen` NavEntry already passes
`onNavigateToRegister` as a direct lambda. The `viewModel` should be
created without the `onLoginSuccess` lambda going through Koin.

**Restructure**: Make `LoginViewModel` store the callback externally, OR
change the factory to take ONLY the repository (no callback) and set the
callback separately:

**Option B ŌĆö Pass callback directly, not via parametersOf:**
Change `LoginViewModel` to not require `onLoginSuccess` in constructor.
Instead expose a `StateFlow<LoginEvent>` that the screen observes:
```kotlin
// PlaybookApp or AppNavigation handles navigation via event
Screen.Login -> NavEntry(screen) {
    val viewModel: LoginViewModel = kmpViewModel()  // no params needed
    LoginScreen(
        viewModel = viewModel,
        onLoginSuccess = { backStack.add(Screen.Events) },  // passed directly
        onNavigateToRegister = { backStack.add(Screen.Register) }
    )
}
```

**This requires:**
1. Remove `onLoginSuccess: () -> Unit` from `LoginViewModel` constructor
2. Expose `val loginSuccess: StateFlow<Boolean>` or use a `Channel`
3. Screen observes and calls navigation callback
4. Same pattern for `RegisterViewModel`

**Option C (simplest) ŌĆö Koin single with explicit type:**
Change `factory { params -> LoginViewModel(get(), onLoginSuccess = params.get()) }`
to `factory { params -> LoginViewModel(get(), onLoginSuccess = params.get<() -> Unit>()) }`
Explicit type parameter might fix the Kotlin Native type resolution.
