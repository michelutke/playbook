## What Was Built

**Task 1 — Offline RSVP sync trigger**

Added `flushQueue()` call inside `LaunchedEffect(authState)` in `TeamorgApp.kt`. When `authState` transitions to `Authenticated` (login or foreground resume after process death), the mutation queue is flushed before the notification polling loop starts. Guard with `hasPendingMutations()` avoids network call when queue is empty. Wrapped in try/catch to silently ignore errors (retry happens next foreground).

**Task 2 — GET /events/{id}/check-in role guard**

Added coach/club_manager role enforcement to the GET check-in handler, mirroring the exact same pattern already used in the PUT handler. Non-coach users now receive `403 Forbidden`. Test added to verify non-coach receives 403. Existing test `check-in entries return empty for event with no team` was fixed to call `setupClubAndTeam()` (which grants `club_manager` role) so it continues to pass after the auth guard.
