**Scenario 1 — ClubManager full setup flow (TM-01, TM-02, TM-06)**
```
fun `clubmanager full setup flow`()
```
Steps:
1. POST /auth/register `{ email, password, displayName }` → 201
2. POST /auth/login → extract JWT token
3. POST /clubs `{ name: "SC Thun", sportType: "volleyball" }` with token → 201, extract clubId
4. POST /clubs/{clubId}/teams `{ name: "U18 Men" }` with token → 201, extract teamId
5. GET /clubs/{clubId}/teams with token → 200, body contains team with name "U18 Men"
6. Assert: response list size >= 1, first team name == "U18 Men"

**Scenario 2 — Invite player via link, new user registers (TM-09, TM-14)**
```
fun `player registers via invite link`()
```
Steps:
1. Register + login as ClubManager, create club + team (use helper)
2. POST /teams/{teamId}/invites `{ role: "player" }` with CM token → 201, extract token from response
3. GET /invites/{inviteToken} (no auth) → 200, body contains teamName, role == "player"
4. Register new user (player), login → extract playerToken
5. POST /invites/{inviteToken}/redeem with playerToken → 200
6. GET /teams/{teamId}/members with CM token → 200, body contains player's userId with role == "player"

**Scenario 3 — Invite coach via link, new user registers (TM-03)**
```
fun `coach registers via invite link`()
```
Same as scenario 2 but with `role: "coach"`. Assert member list contains userId with role == "coach".

**Scenario 4 — Promote player to coach (TM-05)**
```
fun `clubmanager promotes player to coach`()
```
Steps:
1. Register CM, create club + team, invite + register player (role=player)
2. PATCH /teams/{teamId}/members/{playerId}/role `{ role: "coach" }` with CM token → 200
3. GET /teams/{teamId}/members → player's entry now has role == "coach"

**Scenario 5 — Remove coach from team (TM-04)**
