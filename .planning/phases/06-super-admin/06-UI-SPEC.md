## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#090912` | Page background (source: pencil/design.md `--background`) |
| Secondary (30%) | `#13131F` + `#1C1C2E` | Sidebar/nav (`#13131F`), cards and table rows (`#1C1C2E`) (source: pencil/design.md `--surface`, `--card`) |
| Accent (10%) | `#4F8EF7` | Active nav item, primary action buttons, links, focused input ring (source: pencil/design.md `--primary`) |
| Destructive | `#EF4444` | Delete club button, remove ClubManager button, permanent-delete confirmation UI only (source: pencil/design.md `--destructive`) |
| Impersonation | `#F97316` | Impersonation banner background exclusively (source: pencil/design.md `--accent`) |

Accent (`#4F8EF7`) reserved for: active sidebar nav item, all primary action buttons ("Create Club", "Invite ClubManager", "Save Changes"), hyperlinks in audit log, input focus ring. Never used for status indicators or impersonation UI.

Impersonation orange (`#F97316`) reserved for: the impersonation banner only — this ensures it is visually unmissable and distinct from all other interactive elements.

Additional semantic colors (from pencil/design.md, used for status display only):
- `#22C55E` (`--color-success`) — active/reactivated club status badge
- `#FACC15` (`--color-warning`) — pending/deactivated club status badge
- `#EF4444` (`--color-error`) — deleted/error status indicators
- `#9090B0` (`--muted-foreground`) — secondary text, table metadata, timestamps
- `#2A2A40` (`--border`) — table dividers, card borders, input borders
- `#F0F0FF` (`--foreground`) — primary text on dark backgrounds

## Component Inventory

### Layout Shell
- **Sidebar nav** (`#13131F` background, 240px width, fixed): Logo, nav items (Dashboard, Clubs, Users, Audit Log). Active item: `#4F8EF7` text + left border indicator 3px. Hover: `#1C1C2E` bg.
- **Main content area**: `#090912` bg, `xl` (32px) padding, scrollable.
- **Impersonation banner** (44px, `#F97316` bg, full-width, sits above sidebar+content): "Impersonating [Name] — MM:SS remaining" in white semibold + "End Session" button (white outline). Visible on every page during active impersonation. Countdown updates every second client-side.
- **Page header**: Heading (20px/600) + optional breadcrumb in `--muted-foreground`.

### Data Tables
- Row bg: `#1C1C2E`. Header bg: `#13131F`. Header text: 12px/600 `--muted-foreground`. Row text: 14px/400 `--foreground`. Divider: `--border`. Hover row: slight lighten via `opacity-5` white overlay.
- Pagination controls: Previous / Next buttons, "Page N of M" label. Button: `--primary` on active, `--border` on disabled.

### Dashboard Widgets
- Stat card: `#1C1C2E` bg, `lg` (24px) padding. Stat number: 28px/700. Label: 12px/600 `--muted-foreground`. Icon (Lucide, 20px, `--primary`).
- 4-column grid layout on ≥1024px, 2-column on 768–1023px, 1-column on <768px.

### Forms
- Input: `#1C1C2E` bg, `--border` 1px border, `--foreground` text, 14px, height 40px, `md` (16px) horizontal padding. Focus ring: `#4F8EF7` 2px.
- Label: 12px/600 `--foreground`, 4px gap below label.
- Error message: 12px `#EF4444`, 4px gap above.
- Primary button: `#4F8EF7` bg, `#FFFFFF` text, 14px/600, height 40px, `md` padding horizontal, 6px border-radius.
- Destructive button: `#EF4444` bg, `#FFFFFF` text, same sizing.
- Ghost/secondary button: transparent bg, `--border` 1px border, `--foreground` text.

### Confirmation Modals
- `#1C1C2E` bg, `--border` border, `lg` (24px) padding, 8px border-radius.
- Destructive: title in `--foreground` 20px/600, body 14px `--muted-foreground`, type-to-confirm input for permanent delete (see Copywriting), "Delete" button destructive, "Cancel" button ghost.
- Deactivate: same structure, no type-to-confirm, "Deactivate" button uses `#FACC15` (warning) bg with `#090912` text.

### Status Badges
- Active: `#22C55E` text, `rgba(34,197,94,0.12)` bg. 12px/600, `xs` (4px) padding, 4px border-radius.
- Deactivated: `#FACC15` text, `rgba(250,204,21,0.12)` bg. Same sizing.
- Deleted: `#EF4444` text, `rgba(239,68,68,0.12)` bg. Same sizing.

## Interaction States

### Impersonation Flow (SA-08)
1. SA clicks "Impersonate" on a ClubManager row — confirmation modal appears: "Impersonate [Name]? You will act as ClubManager for 1 hour. All actions are audit-logged."
2. On confirm: server issues scoped JWT, orange banner appears at top of all pages with live countdown (updates every second via `setInterval`).
3. SA navigates admin as ClubManager — sees that ClubManager's club context.
4. "End Session" in banner or natural expiry (1h): banner removed, toast shown, page reloads to SA context.
5. If page loaded with expired impersonation token: redirect to `/admin/dashboard` with error toast (see error state above).

### Audit Log (SA-11, SA-12)
- Filterable table: action type dropdown, actor email input, date range (start + end date inputs).
- "Apply Filters" button triggers server-side load function re-fetch.
- "Clear Filters" resets all filters.
- Pagination: 50 rows per page. "Previous" / "Next" / "Page N of M". No infinite scroll.
- Rows are read-only — no edit, no delete affordances anywhere in audit log UI.

### Club Status Transitions
- Active → Deactivate: warning confirmation modal (no type-to-confirm).
- Deactivated → Reactivate: simple confirmation "Reactivate [Club]? Members will regain access immediately." + "Reactivate" button (`#22C55E` bg).
- Any status → Delete: type-to-confirm modal (permanent, irreversible).

### User Search (SA-10)
- Search input at top of Users page: real-time debounce 300ms, min 2 chars.
- Results: table of name, email, clubs, roles, joined date.
- Click row: drawer/panel opens on right showing full profile + club memberships.

## Accessibility

- All interactive elements: min 44×44px click target.
- Focus order: sidebar nav → page header actions → main content.
- All buttons have explicit `type` attribute (`button`, `submit`).
- Destructive buttons: `aria-label` includes action and target name.
- Impersonation banner: `role="alert"` so screen readers announce it on appearance.
- Tables: `<th scope="col">` on all column headers.
- Status badges: also convey meaning via text, not color alone.
