# Requirements: Attendance Tracking

## Overview
Core feature. Every event has an attendance list. Players confirm, decline, or mark themselves as unsure. Coaches get a live overview and can mark attendance manually on-site. Players can view team-wide attendance statistics. Recurring and period absences allow players to pre-declare unavailability.

## Roles
- **Player** — confirms/declines/unsure own attendance, views team attendance & statistics
- **Coach** — views team attendance, marks present/absent manually
- **ClubManager** — views attendance across all club teams

## Functional Requirements

### Player — Per-Event Response
- FR-AT-01: Player can confirm attendance for an event
- FR-AT-02: Player can decline attendance with an optional reason
- FR-AT-03: Player can mark attendance as **unsure** — a Begründung (reason) is mandatory
- FR-AT-04: Player can change their response until the team/club response deadline
- FR-AT-05: Player sees their own attendance history across all events

### Player — Team Visibility & Statistics
- FR-AT-06: Player can view the attendance list of any event (all team members + their status)
- FR-AT-07: Player can view attendance statistics for any team member (e.g. % presence at trainings, % at games)
- FR-AT-08: Statistics filterable by event type (training / match / other) and date range

### Player — Abwesenheit (Pre-declared Absence)
- FR-AT-09: Player can declare a **recurring weekly absence** — select one or more weekdays; optional end date; all events on those days default to `declined-auto`
- FR-AT-10: Player can declare a **period absence** — from date to date (e.g. holidays, injury); all events within that range default to `declined-auto`
- FR-AT-11: Period absence supports an optional label/reason (e.g. "Urlaub", "Verletzung")
- FR-AT-12: Player can view, edit, and delete their active Abwesenheit entries
- FR-AT-13: A manual per-event response always overrides an Abwesenheit default
- FR-AT-14: Coach sees a visual indicator when a decline was auto-generated by an Abwesenheit rule

### Coach — On-Site Check-In
- FR-AT-15: Coach sees all team members and their current response (incl. Begründung for unsure/declined) for an event
- FR-AT-16: Coach can manually override attendance status (`present` / `absent` / `excused`) per player
- FR-AT-17: Coach can add a note to an individual attendance entry

### Attendance Statuses
| Status | Set by | Begründung | Meaning |
|---|---|---|---|
| `confirmed` | Player | Optional | Will attend |
| `declined` | Player | Optional | Will not attend |
| `unsure` | Player | **Mandatory** | Might attend, uncertain |
| `declined-auto` | System | — | Auto-declined via Abwesenheit rule |
| `present` | Coach | — | Physically present |
| `absent` | Coach | — | Didn't show despite confirming |
| `excused` | Coach | — | Excused absence |
| `no-response` | System | — | No action taken |

## Non-Functional Requirements
- Offline: players can submit responses and manage Abwesenheit offline; sync on reconnect
- Response deadline enforced client-side with server validation
- Audit log of manual coach overrides (who changed what, when)
- Statistics queries must be performant for clubs with 100+ members over 2+ years of history

## Out of Scope (MVP)
- J+S Anwesenheitskontrolle exports — data model must support it, but export UI is post-MVP
- GPS / geofence-based automatic check-in
- Biometric attendance
- Integration with external J+S systems

## Resolved
- Response deadline: club/team default setting only (no per-event override)
- Recurring Abwesenheit: optional end date supported
